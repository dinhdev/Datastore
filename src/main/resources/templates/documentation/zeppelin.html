DATAEXO Analysis Platform

DATAEXO gives a unique Analysis Platform which users can use to analyse the business data which they have purchased or got for free. They can send queries and get the desired result for easier understanding of business data.

Users/Customers will use the following link to access the analysis platform http://dataexo.com/data-analysis

To start, first time/new users need to register as per below.  This registration is free of cost and is independent from registering on the main site http://dataexo.com

Authentication is done using API Key:

 

How to get your DATAEXO API Key?

Please sign up at DATAEXO for an absolutely free account. You can purchase either Free data (without any fee) or premium data (Paid)

Free data and Premium data

DATAEXO has an enormous collection of business data including financial data, marketing data, medical and hospital data, real estate data, and many other data sets.

Users/customers can access data on DATAEXO free of cost but with certain limitations.

Fore premium dataset, Users/customers may have to pay DATAEXO. Users can purchase this data either directly or through their subscription. All premium datasets have exhaustive information. If the need arises, the subscription can be cancelled at any point of time.

Although free datasets are suitable for testing, trialling and study, yet we strongly suggest usage of premium datasets for proficient and professional purposes.

 

To get API Key, login to http://www.dataexo.com

Click on Account Settings

 

You will be directed to the Profile page directly

 

You can also access profile page by going to http://www.dataexo.com/account/mydata

 

  • Click on Edit profile
  • Get your API Key from profile page

 

  • You will be using this API Key while registering to analyze platform dataexo.com/data-analysis
  • Put the User name and Email which you used while registering on DATAEXO- dataexo.com
  • Insert the API key which you got while purchasing your datasets and resgister.

  • Click on Create a new note
  • Give it a name synonym with the query you are going to write for easier identification.
  • You can also write a brief description about the query and the expected output.
